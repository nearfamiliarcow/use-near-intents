---
name: Content and Component Decisions
description: Key product decisions about how to build the showcase site — screenshots over demos, visual storytelling priority
type: feedback
---

Real screenshots are preferred over demo components (SwapDemo, CrossPayDemo, OnboardingDemo).

**Why:** Demo components were deprioritized in favor of real screenshots, which are more credible and already exist for ZODL. The site already has 6 real ZODL screenshots with captions.

**How to apply:** When designing new components, build around real screenshots/images as the visual anchor. Demo components are a future enhancement, not a dependency. Don't block visual improvement work on demo components not existing yet.

---

Use case detail pages are text-heavy and need more visual work. Reusable components from case studies should serve both case study pages AND use case pages.

**Why:** The use case pages currently have no screenshots or visual storytelling. The goal is to pull proven components from case studies into use case pages to make them feel more alive.

**How to apply:** Design every new component with dual placement in mind — it should work on both the case study detail page AND be embeddable in use case detail pages.

---

The site philosophy is "show not tell" — visual components that demonstrate what the integration does are more valuable than text descriptions.

**How to apply:** When choosing between a text block and a visual component, default to the visual. Metrics, before/after comparisons, flow diagrams, and screenshots always outperform prose.
